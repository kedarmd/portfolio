# AGENTS.md

This guide is for agentic coding tools working in this repo.
Keep changes consistent with existing patterns and avoid introducing new toolchains.

## Project snapshot
- Stack: Fresh v2 (JSR) + Preact + Tailwind CSS v4
- Runtime: Deno (no Node/Vite)
- Static dir: `static/`
- Source roots: `routes/`, `components/` (add `islands/` only when needed)

## Commands
Dev server:
- `deno task dev`

Build production bundle:
- `deno task build`

Start production server (after build):
- `deno task start`

Update Fresh tooling:
- `deno task update`

Format:
- `deno fmt`

Lint:
- `deno lint`

### Tests
- No test runner configured yet.
- When tests are added, prefer `deno test path/to/test.ts` for single test files.

## Cursor/Copilot rules
- No `.cursor/rules/`, `.cursorrules`, or `.github/copilot-instructions.md` found.

## Code structure
- `routes/`: page routes and layout (_app, _layout, _404)
- `components/`: shared UI components
- `static/`: static assets and global CSS (`static/styles.css`)
- `dev.ts`: Fresh builder entry (Tailwind plugin)
- `main.ts`: Fresh app entry (App + fsRoutes)
- `utils.ts`: shared `define` helpers and `State` type

## Routing conventions (Fresh)
- Each route file exports a default component.
- `_app.tsx` provides global `<Head>` metadata and stylesheet link.
- `_layout.tsx` wraps all routes with shared layout (Navbar, page shell).
- `_404.tsx` handles not-found UI.
- Routes are path-based: `routes/index.tsx` -> `/`.

## Islands and interactivity
- Keep routes SSR by default.
- Move interactive UI to `islands/` and keep islands small.
- Avoid hooks in `routes/` or shared SSR components.

## TypeScript and Preact style
- Use function components only.
- Prefer function declarations for components and helpers.
- Use `type` imports for types (e.g., `import type { ComponentChildren } from "preact"`).
- Avoid `any`; keep types precise and scoped.
- Use `const` for immutable data tables.

## Imports
- Order: external imports first, then local modules.
- Use explicit `.ts`/`.tsx` extensions for local imports.
- Keep a blank line between import groups.

## Naming
- Components: `PascalCase` file and function names (e.g., `Navbar`).
- Routes: lowercase filenames matching paths (e.g., `experience.tsx`).
- Constants: `UPPER_SNAKE_CASE` for static arrays/objects.
- Functions: `camelCase`.

## Formatting
- Indentation: 2 spaces.
- Quotes: double quotes for strings.
- Semicolons: always.
- Trailing commas: use in multiline objects/arrays/arguments.
- JSX: wrap long props and keep formatting aligned.

## Styling
- Tailwind utilities for layout and spacing.
- Global theme tokens live in `static/styles.css` as CSS variables.
- Tokyo Night palette must be preserved (see `:root` in `static/styles.css`).
- Inline styles are acceptable for CSS variables (e.g., `style={{ color: "var(--color-text)" }}`)
- Add new global CSS sparingly and keep it in `static/styles.css`.

## Error handling and UX
- Prefer guard clauses and explicit fallbacks.
- Use the shared `NotFound` component for 404 experiences.
- Avoid throwing uncaught errors in UI render paths.

## File generation and automation
- Fresh build artifacts live in `_fresh/` (generated by `deno task build`).

## Adding new features
- Add new pages as route files in `routes/`.
- For client-only behavior, move UI to `islands/` and keep islands small.
- Put shared UI in `components/` and keep components small.
- Update `components/Navbar.tsx` if you add top-level pages.

## Static assets
- Place images and static files in `static/`.
- Reference them with absolute paths (e.g., `/avatar.jpg`).

## Safety and hygiene
- Do not add Node/Vite tooling unless explicitly requested.
- Avoid touching build artifacts (`dist/`, `.output/`, `.vite/`, `.tanstack/`, `_fresh/`).
- Keep dependencies minimal and aligned with Deno/Fresh.
